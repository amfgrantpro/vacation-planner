# **Project Brief: Agentic tool**

## **1\. Purpose of this Project**

This is a **solo, vibe-coded exploration project** intended to ground my understanding of how modern AI products (B2B and B2C) are built and shipped in 2026\.

The goal is **learning through building** \- using strong product judgment to select, adapt, and combine best‑in‑class UX patterns that are feasible for a single person to implement with an AI coding agent (VS Code).

This is not a startup pitch, a venture-scale platform, or an attempt to invent a new product.

It is a practical product exploration designed to:

* Mirror how companies are actually building AI products today  
* Extend my vibe-coding experience into new areas of AI technology.

---

## **2\. Who is building this**

This project will be built by a single person. They are an experienced Senior Product Manager. They are not a full-time engineer.

Relevant prior experience:

* Experienced PM in both B2B and B2C products.  
* Comfortable with modern UX concepts and system thinking.

  * Modern UX flows  
  * Accessibility focus  
  * Text \+ file input  
  * Multi-shot prompting  
  * Real end-to-end working behavior  
  * Gradio demo, Loveable UI, Personally-designed UX, Vercel deployment

* Remote deployment not a large consideration at this stage

---

## **3\. Product space: Agentic AI**

This project scope has been decided because of a pattern observed through job interviews and industry exposure. Companies like Doctolib, Grammarly, and Contentful are shipping **agentic systems**.

* The user-facing experience often looks simple or familiar to their core product experience.  
* The complexity is hidden in task orchestration, reasoning boundaries, artifact generation and iteration handling.  
* The end-user is not using “AI tools” — they are doing their usual work, with agentic capabilities embedded into the flow.

This project is meant to build intuition for agentic and broader AI-product system designs. I hope to gain **first-hand experience** with:

* Designing agent-centered workflows  
* Understanding trade-offs between chat, tools, agents, and their impacts on UX and CX  
* Understanding how multiple reasoning models coexist inside one product  
* Understanding how AI outputs can be converted into UX for end-user task success and enjoyment.

---

## **4\. Success criteria**

This project is successful if:

* it produces a working, coherent tool  
* the UX feels like it aids in task success  
* the system helps users make *real decisions*  
* the builder gains concrete insight into agentic product design

Shipping something small and correct is valued more than shipping something broad.
